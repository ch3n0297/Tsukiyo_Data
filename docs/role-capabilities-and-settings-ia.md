# 角色能力與設定資訊架構

## 1. 文件目的

本文件整理社群資料中台前端的角色能力邊界，以及設定模組應如何拆分頁面。

目標有三個：

- 明確區分 `member` 與 `admin` 可以做什麼
- 避免把「管理別人」與「設定自己」混在同一頁
- 提供 `.pen` 設計稿與後續前端實作可共用的資訊架構

## 2. 核心原則

### 2.1 角色分工

- `member`
  - 可查看資料
  - 可管理自己的偏好與帳號安全
  - 不可管理其他使用者
  - 不可變更系統層設定
- `admin`
  - 擁有 `member` 的全部能力
  - 另外可管理使用者、審核註冊、查看系統層設定
  - 可檢視平台接入、排程與治理資訊

### 2.2 設定頁拆分原則

- `個人設定` 是「我設定自己」
- `使用者管理` 是「我管理別人」
- `系統設定` 是「我管理整個系統」

這三者不能混在同一頁，否則資訊架構會失焦。

## 4. `member` 預期能做的事

一般帳號不是只有「登入後看資料」，至少應該有以下能力：

### 4.1 查看與追蹤

- 查看總覽看板
- 查看帳號列表
- 查看單一帳號詳情
- 查看最新輸出與同步狀態
- 查看異常摘要與系統健康度

### 4.2 個人化

- 切換語言
- 切換主題
  - 跟隨系統
  - 淺色
  - 深色
- 設定時區
- 設定日期時間格式
- 設定登入後預設首頁

### 4.3 帳號安全

- 修改顯示名稱
- 修改 Email
- 修改密碼
- 查看目前登入狀態
- 登出
- 登出其他裝置

### 4.4 通知

- 是否接收同步失敗通知
- 是否接收每日摘要
- 通知頻率

## 5. 設定模組資訊架構

### 5.1 所有人可見

#### `Settings / 個人設定`

定位：

- 一般使用者與管理員共用
- 專注在個人偏好與帳號安全

內容應包含：

- 個人資料卡
  - 顯示名稱
  - Email
  - 角色
  - 最後登入時間
- 介面偏好
  - 語言
  - 主題模式
  - 時區
  - 日期時間格式
  - 預設首頁
- 安全設定
  - 修改密碼
  - 目前 session
  - 登出其他裝置
- 通知偏好
  - 同步失敗通知
  - 審核結果通知
  - 摘要通知

### 5.2 僅限 admin

#### `Admin / 使用者管理`

定位：

- 管理使用者生命週期
- 不處理系統排程或平台接入

內容應包含：

- 使用者總數
- 啟用中 / 待審 / 停用統計
- 使用者清單
  - 顯示名稱
  - Email
  - 角色
  - 狀態
  - 建立時間
  - 最後登入
- 管理操作
  - 核准
  - 拒絕
  - 停用
  - 重新啟用
  - 角色調整

#### `Admin / 待審註冊申請`

定位：

- 專注處理待審核註冊流
- 作為 `使用者管理` 的聚焦工作台

#### `Settings / 平台授權與帳號接入`

定位：

- 檢視平台接入狀態、授權健康度與帳號接入資訊

#### `Settings / 排程與系統設定`

定位：

- 檢視排程器、併行數、保護規則與系統運作參數
- 首版偏資訊型，不預設直接可寫入

## 6. 建議導航結構

### 6.1 member 導航

- `總覽看板`
- `資料來源`
- `同步任務`
- `個人設定`

### 6.2 admin 導航

- `總覽看板`
- `資料來源`
- `同步任務`
- `個人設定`
- `使用者管理`
- `管理員審核`
- `帳號接入`
- `系統設定`

## 7. 對 `.pen` 設計稿的落地建議

### 7.1 立即調整

- 新增 `Settings / 個人設定`
- 將既有 `Settings / 帳號設定管理` 改為 `Admin / 使用者管理`
- 保留 `Admin / 待審註冊申請`
- 保留 `Settings / 排程與系統設定`

### 7.2 後續可補

- 在 `Settings / 個人設定` 補上語言與主題切換
- 在 admin 導航中補上更清楚的角色區隔
- 為 `member` 與 `admin` 各畫一份 sidebar 變體

## 8. 命名建議

- 不建議再使用 `帳號設定管理` 這種易混淆名稱
- 若是管理別人的帳號，請用 `使用者管理`
- 若是設定自己的帳號，請用 `個人設定`
- 若是管理系統規則，請用 `系統設定`
